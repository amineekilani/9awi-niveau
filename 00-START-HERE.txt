🔄 FLUX D'UPLOAD:

  1. Utilisateur sélectionne image
  2. Prévisualisation affichée
  3. Validation (type, taille)
  4. Upload via FormData
  5. Backend reçoit MultipartFile
  6. Génère UUID comme nom
  7. Sauvegarde dans uploads/users/
  8. Stocke nom en BD
  9. Retourne URL
  10. Frontend affiche image

═══════════════════════════════════════════════════════════════════════════════════

📊 ENDPOINTS API:

  Public:
    POST /images/users/upload
    GET /images/users/{filename}

  Authentifié:
    POST /api/profile/upload-image (JWT)
    GET /api/profile (JWT) → inclut profileImage

🐛 TROUBLESHOOTING:

  Dossier uploads n'existe pas?
    → Créé automatiquement au premier upload

  Erreur 413 (trop volumineux)?
    → Augmentez les limites dans application.properties

  Image ne s'affiche pas?
    → Vérifiez: serveur lancé + fichier existe + URL correcte

  Erreur 401 lors upload profil?
    → Vérifiez que le JWT token est valide
