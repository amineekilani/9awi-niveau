CORRECTIONS APPLIQUEES :
------------------------

1. ✅ CACHER LES QUESTIONS POUR LES ETUDIANTS
   Fichier : frontend/src/app/module-detail/module-detail.html
   
   Avant :
   <div *ngIf="quiz.questions && quiz.questions.length > 0">
   
   Après :
   <div *ngIf="authService.isFormateur() && quiz.questions && quiz.questions.length > 0">
   
   → Les questions ne sont visibles que pour les formateurs
   → Les étudiants voient seulement le bouton "Passer le quiz"

2. ✅ BLOQUER LE QUIZ SI LECONS NON COMPLETEES
   Fichier : frontend/src/app/module-detail/module-detail.ts
   
   Ajout de la méthode :
   allLeconsCompleted(): boolean {
     if (this.lecons.length === 0) return false;
     return this.lecons.every(lecon => lecon.completed);
   }
   
   Fichier : frontend/src/app/module-detail/module-detail.html
   
   Bouton actif si toutes les leçons complétées :
   <a *ngIf="allLeconsCompleted()" [routerLink]="...">Passer le quiz</a>
   
   Bouton désactivé sinon :
   <button *ngIf="!allLeconsCompleted()" disabled>Quiz verrouillé 🔒</button>
   
   Message d'avertissement :
   "⚠️ Complétez toutes les leçons pour débloquer le quiz"

3. ✅ DEBUG DES OPTIONS
   Fichier : frontend/src/app/quiz-viewer/quiz-viewer.ts
   
   Ajout de console.log pour débugger :
   - Quiz chargé
   - Questions
   - Options de chaque question
   
   Fichier : frontend/src/app/quiz-viewer/quiz-viewer.html
   
   Ajout d'un message si pas d'options :
   "⚠️ Aucune option disponible pour cette question"

POUR TESTER :
-------------

1. REDEMARRER LE FRONTEND :
   Ctrl+C dans le terminal frontend
   npm start

2. TEST ETUDIANT :
   a) Se connecter en tant qu'étudiant
   b) Ouvrir un module avec quiz
   c) Vérifier que les questions ne sont PAS visibles
   d) Vérifier que le bouton est "Quiz verrouillé"
   e) Compléter TOUTES les leçons
   f) Vérifier que le bouton devient "Passer le quiz"
   g) Cliquer sur "Passer le quiz"
   h) Vérifier que les OPTIONS s'affichent
   i) Répondre et valider

3. VERIFIER LA CONSOLE :
   F12 → Console
   Chercher les logs :
   - "Quiz chargé:"
   - "Questions:"
   - "Options de la première question:"
   
   Si les options sont vides [], c'est un problème backend

================================================================================

SI LES OPTIONS NE S'AFFICHENT TOUJOURS PAS :
---------------------------------------------

1. Ouvrir la console (F12)
2. Regarder les logs
3. Si "Options: []" → Problème backend
4. Vérifier que le formateur a bien ajouté des options aux questions
5. Vérifier dans la base de données :
   SELECT * FROM question WHERE quiz_id = X;
   
6. Les options doivent être stockées comme JSON array

================================================================================

VERIFICATION BACKEND :
----------------------

Si les options sont vides, vérifier :

1. Table question :
   - Colonne "options" de type TEXT ou JSON
   - Contient bien un array JSON : ["Option A", "Option B", "Option C"]

2. QuestionResponse.java :
   - private List<String> options; ✅

3. Lors de la création d'une question :
   - Les options sont bien sauvegardées
   - Le JSON est bien formé
